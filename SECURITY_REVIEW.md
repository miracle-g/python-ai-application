### [Low] 3. 依存パッケージのバージョンが未固定

- **場所**: `requirements.txt`（`streamlit>=1.40` / `google-genai>=1.0` / `python-dotenv>=1.0`）
- **何が問題か**: 下限のみ指定で上限・固定がなく、ロックファイルもありません。意図しない新バージョンや
  供給網侵害バージョンが入る余地があります。
- **想定シナリオ / 影響**: 再現性低下とまれな悪性バージョン混入。個人ツールでは Low。
  **`pip-audit` が未導入のため、既知CVEの自動確認は未実施**です。
- **修正方法**: `pip freeze > requirements.lock` で固定、または `==` 指定。一度
  `pip install pip-audit && pip-audit -r requirements.txt` で既知脆弱性を確認しておくとよい。

## 良かった点

この構成で事故りやすいポイントの多くが、既に正しく守れています。今後の変更でここを壊さないよう
意識してください。

- **旧キーの再発行（失効）を実施済み** — 露出したキーへの一次対応として正しい判断（あとは
  置き場所を直せば完了）。
- **ソースにキーのハードコードなし**。APIキー入力欄は `type="password"`。
- **キー解決順が堅牢**: `session_state → 環境変数 → st.secrets`。secrets.toml 不在時の例外も
  `try/except` で握られている。
- **XSS の起点がない**: 生成結果は `st.markdown` / `st.write_stream` で表示、HTMLは無効。
  `unsafe_allow_html` も `st.html` も未使用。
- **危険な動的実行なし**: `eval`/`exec`/`os.system`/`subprocess`/`pickle.load` を使っていない。
- **パス操作が安全**: ダウンロード名は `_safe_filename()` でサニタイズ、履歴名も固定要素のみ。
  サーバ側で入力由来のパスを `open()` していない。
- **`.gitignore` が要所を押さえている**: `.env` / `data/` / `secrets.toml` / `__pycache__` を除外。
  （穴は `.env.example` のみ → 発見#1）
